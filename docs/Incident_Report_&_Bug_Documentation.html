3. Laporan Insiden & Dokumentasi Bug Hasil Analisis Kode

Berikut adalah laporan insiden dan bug nyata yang berhasil ditarik dari hasil analisis kode dan foto-foto dari Google Play Store serta hasil uji OCR.

🚨 Laporan Insiden: Kegagalan Rilis Aplikasi ke Google Play Store

  • ID Insiden: INC-20260712-01
  • Tanggal Kejadian: 12/07/2026
  • Skala Keparahan (Severity): Critical
  • Dampak Layanan: Aplikasi mobile Medika tidak dapat didistribusikan ke Google Play Store (proses rilis terhambat total).
  • Analisis Akar Masalah (RCA):
    • Masalah 1 (Debug Signature): Berkas bundle app-release.aab dibangun menggunakan tanda tangan debug default Flutter, yang ditolak otomatis oleh Google Play Console untuk alasan keamanan rilis.
    • Masalah 2 (Package Name Mismatch): Konfigurasi applicationId pada file gradle masih berupa default com.example.apotek_web, sedangkan aplikasi terdaftar di Play Console dengan ID com.hn.medika.
  • Tindakan Perbaikan:
    1. Membuat file keystore rilis aman (upload-keystore.jks) dan memetakan kredensial di key.properties.
    2. Memperbarui konfigurasi Gradle untuk memproses penandatanganan rilis secara dinamis.
    3. Mengubah applicationId serta memindahkan class MainActivity.kt ke package name yang sesuai (com.hn.medika).

🐛 Dokumentasi Bug yang Diketahui

BUG-001: Penolakan Kunci Tanda Tangan Debug di Play Store

  • Komponen Terkait: Android Build & Deployment
  • Skala Keparahan (Severity): Critical
  • Langkah untuk Mereproduksi:
    1. Jalankan perintah flutter build appbundle --release di lokal.
    2. Unggah berkas app-release.aab yang dihasilkan ke Google Play Console.
  • Ekspektasi vs Realita:
    • Ekspektasi: Google Play Console berhasil menerima bundel aplikasi untuk didistribusikan ke pengguna.
    • Realita: Google Play Console memunculkan galat bahwa file AAB ditandatangani menggunakan sertifikat pengujian (debug mode).
  • Solusi / Perubahan Kode:

    Membuat berkas key.properties untuk menyimpan kredensial rilis dan mengubah build.gradle.kts:

    signingConfigs {
    create("release") {
        keyAlias = properties["keyAlias"] as String
        keyPassword = properties["keyPassword"] as String
        storeFile = file(properties["storeFile"] as String)
        storePassword = properties["storePassword"] as String
    }
}
buildTypes {
    release {
        signingConfig = signingConfigs.getByName("release")
    }
}

BUG-002: Ketidakcocokan Nama Paket (Application ID)

  • Komponen Terkait: Android Manifest / Build Configuration
  • Skala Keparahan (Severity): Critical
  • Langkah untuk Mereproduksi:
    1. Bangun berkas rilis AAB bertandatangan.
    2. Coba unggah ke dalam aplikasi Google Play Console milik com.hn.medika.
  • Ekspektasi vs Realita:
    • Ekspektasi: ID paket cocok dan proses pengunggahan berhasil.
    • Realita: Terjadi penolakan dengan pesan galat: "APK atau Android App Bundle Anda harus memiliki nama paket com.hn.medika."
  • Solusi / Perubahan Kode:

    Ubah penamaan namespace dan applicationId di file android/app/build.gradle.kts:

    - namespace = "com.example.apotek_web"
- applicationId = "com.example.apotek_web"
+ namespace = "com.hn.medika"
+ applicationId = "com.hn.medika"

    Latu sesuaikan baris pertama MainActivity.kt menjadi package com.hn.medika.

BUG-003: Fitur Scan OCR Resep Gagal Membaca Tulisan Tangan (Silent Failure)

  • Komponen Terkait: Mobile OCR & UX
  • Skala Keparahan (Severity): High
  • Langkah untuk Mereproduksi:
    1. Buka fitur unggah resep dokter di aplikasi mobile pasien.
    2. Pilih gambar resep dokter dengan tulisan tangan sambung/miring (seperti teks "Zitoma" / "Azitoma").
    3. Klik tombol Scan & Proses Resep.
  • Ekspektasi vs Realita:
    • Ekspektasi: Hasil pindaian obat ditampilkan secara akurat di layar, atau aplikasi memberikan feedback berupa notifikasi SnackBar jika teks tidak terbaca sama sekali.
    • Realita: Indikator loading berputar sejenak lalu menghilang tanpa ada pesan kesalahan maupun hasil teks yang keluar (silent failure).
  • Solusi / Perubahan Kode:

    Menghapus pustaka lokal ML Kit (yang dirancang hanya untuk teks cetak) dan memindahkan logika OCR resep ke endpoint backend /external/ocr-prescription yang memproses model ONNX terlatih (ocr_prescription_model.onnx) dengan koreksi Levenshtein Distance.

    Menambahkan SnackBar feedback di sisi mobile:

    if (ocrResult.isEmpty) {
    ScaffoldMessenger.of(context).showSnackBar(
        SnackBar(content: Text('Resep tulisan tangan tidak dapat dibaca. Silakan foto kembali resep Anda secara tegak dan jelas.')),
    );
}

BUG-004: Kegagalan Kompilasi Kotlin di Windows (Drive Root Mismatch)

  • Komponen Terkait: Local Developer Environment (Build Tooling)
  • Skala Keparahan (Severity): High
  • Langkah untuk Mereproduksi:
    1. Simpan direktori proyek di Drive D:\ pada sistem operasi Windows.
    2. Pastikan folder cache pub berada di default C:\Users\....
    3. Jalankan flutter build appbundle dari terminal proyek.
  • Ekspektasi vs Realita:
    • Ekspektasi: Aplikasi terkompilasi dengan lancar.
    • Realita: Build gagal dengan galat fatal: IllegalArgumentException: this and base files have different roots dari kompiler Kotlin saat mengompilasi berkas image_picker_android.
  • Solusi / Perubahan Kode:

    Menghapus pustaka native yang tidak lagi digunakan (google_mlkit_text_recognition) karena OCR telah didelegasikan ke backend. Masalah cross-drive compiler dapat dihindari dengan memindahkan sementara folder proyek ke Drive C:\ atau mengatur ulang environment variable cache pub agar seragam di Drive D:\`.

BUG-005: Hang Tanpa Batas Saat Login di HP Samsung (Keystore Cloud Restore Bug)

  • Komponen Terkait: Authentication & Secure Storage
  • Skala Keparahan (Severity): Critical
  • Langkah untuk Mereproduksi:
    1. Pasang aplikasi di HP Samsung, jalankan login, dan biarkan sistem mencadangkan data ke cloud Samsung/Knox.
    2. Lakukan uninstal aplikasi.
    3. Unduh kembali aplikasi dari cloud backup, pasang ulang, lalu buka aplikasi dan coba login kembali.
  • Ekspektasi vs Realita:
    • Ekspektasi: Token lama terhapus bersih secara otomatis, aplikasi membuka halaman login bersih.
    • Realita: Aplikasi terkunci di splash screen/halaman login berputar selamanya karena data token lama berhasil dipulihkan secara otomatis oleh cloud, tetapi kunci dekripsi Keystore yang digunakan untuk membacanya telah dihancurkan oleh sistem saat uninstal, memicu deadlock thread.
  • Solusi / Perubahan Kode:

    Aktifkan parameter encryptedSharedPreferences pada modul secure storage Android, serta terapkan timeout dan catch blok untuk meriset penyimpanan (self-healing):

    try {
    final token = await secureStorage.read(key: 'auth_token').timeout(Duration(seconds: 2));
} catch (e) {
    // Hapus total data secure storage yang korup/tidak bisa didekripsi
    await secureStorage.deleteAll();
}

BUG-006: PlatformException (Channel Error) Kamera di Versi Mobile Rilis

  • Komponen Terkait: Flutter Platform Channel (Native Bindings)
  • Skala Keparahan (Severity): High
  • Langkah untuk Mereproduksi:
    1. Bangun rilis APK dan instal di ponsel fisik.
    2. Buka menu unggah resep, lalu tekan tombol memicu kamera.
  • Ekspektasi vs Realita:
    • Ekspektasi: Kamera internal ponsel terbuka secara responsif.
    • Realita: Aplikasi crash seketika dan melempar kesalahan: PlatformException(channel-error, ...).
  • Solusi / Perubahan Kode:

    Memindahkan library native (image_picker, permission_handler) dari bagian dev_dependencies ke dependencies utama pada file pubspec.yaml agar modul native bindings terdaftar dengan tepat ke dalam struktur build APK rilis.
